# Linux `ls` Command

The `ls` command is one of the most frequently used Linux commands.

It is used to:

- List files
- List directories
- View hidden files
- Check permissions
- View file sizes
- Display inode numbers
- Sort files

As a Linux administrator, you will use `ls` hundreds of times every day.

---

# What does LS mean?

`ls` stands for:

```text
List
```

---

# Syntax

```bash
ls [OPTIONS] [FILE/DIRECTORY]
```

---

# Basic Usage

```bash
ls
```

Example:

```text
Documents Downloads Pictures Music Videos
```

---

# Listing Specific Directory

```bash
ls /etc
```

```bash
ls /var/log
```

```bash
ls /home
```

---

# Command Options

| Flag | Description |
|------|-------------|
| `-a` | Show hidden files |
| `-l` | Long listing format |
| `-h` | Human readable sizes |
| `-la` | Long listing + hidden files |
| `-R` | Recursive listing |
| `-t` | Sort by modification time |
| `-S` | Sort by size |
| `-r` | Reverse order |
| `-i` | Show inode numbers |
| `-d` | Show directory itself |
| `-F` | Classify file types |
| `--color` | Colored output |
| `--help` | Display help |

---

# `ls -a`

Shows hidden files.

Linux hidden files begin with:

```text
.
```

Example:

```bash
ls -a
```

Output:

```text
.
..
.bashrc
.profile
.ssh
Documents
Downloads
```

---

# Why Hidden Files Matter

Hidden files usually contain:

- User configurations
- Git settings
- SSH keys
- Application settings

Examples:

```text
.bashrc
.gitconfig
.ssh
.env
```

---

# `ls -l`

Displays detailed information.

Example:

```bash
ls -l
```

Output:

```text
drwxr-xr-x 2 aayan users 4096 Jul 14 Documents
-rw-r--r-- 1 aayan users 1542 Jul 14 notes.txt
```

---

# Understanding `ls -l`

Example:

```text
drwxr-xr-x
```

---

## First Character

| Symbol | Meaning |
|---------|----------|
| `d` | Directory |
| `-` | Regular File |
| `l` | Symbolic Link |
| `c` | Character Device |
| `b` | Block Device |

---

## Permissions

```text
rwxr-xr-x
```

Breakdown:

```text
rwx → Owner
r-x → Group
r-x → Others
```

---

## Link Count

```text
2
```

Number of hard links.

---

## Owner

```text
aayan
```

---

## Group

```text
users
```

---

## File Size

```text
4096
```

Measured in bytes.

---

## Date

```text
Jul 14
```

Last modification date.

---

## Filename

```text
Documents
```

---

# `ls -h`

Makes sizes easier to read.

Example:

```bash
ls -lh
```

Output:

```text
1.2K file.txt
500M backup.zip
2.4G world.tar.gz
```

Without:

```bash
ls -l
```

Output:

```text
2534763492 world.tar.gz
```

---

# `ls -la`

Most commonly used option.

Example:

```bash
ls -la
```

Shows:

✅ Hidden files

✅ Permissions

✅ Ownership

✅ Sizes

---

# `ls -R`

Recursive listing.

Lists all files inside subdirectories.

Example:

```bash
ls -R
```

Output:

```text
Projects:
website
server

website:
index.php
style.css
```

---

# `ls -t`

Sort by modification time.

Newest files appear first.

Example:

```bash
ls -lt
```

Useful for:

- Logs
- Server debugging
- Development

---

# `ls -S`

Sort by file size.

Example:

```bash
ls -lhS
```

Largest files appear first.

Useful for:

- Finding huge backups
- Identifying large world folders

---

# `ls -r`

Reverse sorting.

Example:

```bash
ls -lr
```

---

# `ls -i`

Display inode numbers.

Example:

```bash
ls -i
```

Output:

```text
45123 file.txt
45124 backup.zip
```

Useful for:

- Filesystem debugging
- Hard links
- Forensics

---

# `ls -d`

Shows directory itself.

Example:

```bash
ls -ld /etc
```

Output:

```text
drwxr-xr-x root root /etc
```

---

# `ls -F`

Classifies file types.

Example:

```bash
ls -F
```

Output:

```text
Documents/
script.sh*
shortcut@
```

Meaning:

| Symbol | Meaning |
|---------|----------|
| `/` | Directory |
| `*` | Executable |
| `@` | Symbolic Link |

---

# Combining Flags

Linux allows multiple options.

Example:

```bash
ls -lah
```

Equivalent:

```bash
ls -l -a -h
```

```bash

```

